= API Gateway Service

== Overview

The API Gateway serves as the central entry point for the Wordle microservices architecture. It handles authentication, request routing, CORS configuration, and service coordination between the frontend and backend services.

== Features

=== Request Routing
- User authentication: /api/users/* → User Service
- Game operations: /api/game/* → Game Service
- Profile management: /api/profile/* → Profile Service

=== Authentication
- JWT token validation for production
- GitLab OAuth2 integration
- Automatic auth context injection

=== Cross-Service Features
- CORS configuration for web clients
- Request/response logging
- Error handling and service fallbacks
- Health monitoring
